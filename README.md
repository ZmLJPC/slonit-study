# Команды Git
---
Ранее я ознакомился со следующими командами Git, благодаря [этому плейлисту](https://www.youtube.com/playlist?list=PLDyvV36pndZFHXjXuwA_NywNrVQO0aQqb)
- `git add <file>` - команда для добавления файла/изменения в **index** *(`.` для добавления всех файлов/изменений в index)*
    - `-f, --force` - общий флаг для принудительного добавления файла/измениня в **index**
    - `-A` - флаг для добавления всех изменений, начиная с **корня** проекта
    - `-p` - флаг для добавления конкретных изменений в файле в **index**
- `git show <commit>` - отображает информацию об указанном коммите *(если коммит не указан, показывает информацию о текущем коммите)*
- `git commit` - команда для сохранения изменений из **index** в **репозиторий**
    - `-m` - флаг для добавления сообщения к коммиту
    - `-a, -am` - флаг, который одновременно выполняеи и `git add .`, и `git commit` *(флаг `-am` также принимает и сообщение коммиту)*
- `git config` - команда для работы с конфигом Git
    - `--system, --global, --local` - флаги для обозначения, в какой конфиг будут внесены изменения *(системный, пользователя или локальный)*

Для загрузки этого файла в Github, с локального репозитория на удалённый, были использованы следующие команды:
```bash
git init
git add README.md
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/ZmLJPC/slonit-study.git
git push -u origin main
```
